# Dreaming and Palace

## Roles

### Draft rooms

Draft rooms preserve process. They are useful for traceability, but they should not be the default input for high-level memory synthesis.

### Curated rooms

Curated rooms contain the material that should be easy to recall later. These are the best first inputs for a palace or wiki layer.

### Dreaming

Dreaming is a summarization and promotion aid, not the primary source of truth. It can surface patterns and candidate truths, but it should not replace the curated memory structure.

### Palace or wiki layer

The palace/wiki layer is a map, not the raw archive.

It should preferably ingest:

1. reflections
2. relationships
3. preferences
4. open questions

and only later, if intentionally chosen:

5. drafts

## Recommended cooperation pattern

1. Raw process lands in draft rooms.
2. The agent writes compact settled notes into curated rooms.
3. Dreaming may observe the broader memory environment.
4. Palace/wiki sources should be synced from curated rooms first.
5. Retrieval should prefer curated rooms over draft rooms.

## What usually goes wrong

- Draft rooms are indexed as if they were final memory.
- Palace/wiki layers ingest everything and become noisy.
- Dreaming is treated as the long-term memory authority.
- Open questions are flattened into fake conclusions.

## Healthy boundary

Think of the system like this:

- drafts preserve how the agent got there
- curated rooms preserve what should remain easy to find
- dreaming notices patterns
- palace/wiki layers organize the cleaned result
